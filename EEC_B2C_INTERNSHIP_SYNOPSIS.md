# Internship Synopsis

## 1. Title of the Project
EEC B2C: AI-Enabled Role-Based Learning and Assessment Platform

## 2. Internship Details
- Internship Domain: Full Stack Web Development (EdTech)
- Project Type: Industry-oriented B2C Educational Platform
- Technology Nature: Web-based, role-driven, modular, scalable

## 3. Introduction
The education technology ecosystem requires platforms that can combine learning content, testing, communication, and administration within one secure and scalable system. In many institutes, these functions are still distributed across disconnected tools, resulting in inefficiency and poor user experience.

The EEC B2C project was developed to address this gap by building a complete digital learning platform with role-specific access for students, teachers, and administrators. The platform integrates assessments, AI-powered question workflows, paid learning resources, subscription models, and communication features into a unified application.

## 4. Problem Statement
Traditional learning systems face the following limitations:
- Lack of centralized workflows for content, exams, and student progress.
- Limited personalization and poor feedback cycles for learners.
- Absence of scalable monetization modules for modern digital learning businesses.
- Weak integration of automation and AI-assisted educational processes.

Therefore, a robust and integrated B2C learning platform was required to ensure better learning delivery, operational efficiency, and business sustainability.

## 5. Objectives
The main objectives of the internship project were:
- To design and implement a role-based educational platform for Public, Student, Teacher, and Admin users.
- To enable secure authentication, user profile management, and permission-controlled operations.
- To develop modules for question management, exams, results, and learning analytics.
- To integrate AI-driven question generation from PDF content.
- To implement paid features such as study material purchase and subscription packages.
- To provide engagement features including notifications, push alerts, chat, and gamified activities.

## 6. Scope of Work
During the internship, the following areas were addressed:
- Frontend development of public pages and dashboard interfaces.
- Backend API design for all functional modules.
- Database schema implementation and role-based data flow control.
- Third-party integration (Razorpay, Cloudinary, Email, Web Push).
- AI service integration for automated question generation.
- End-to-end testing of major user journeys and module connectivity.

## 7. System Methodology
The project followed a modular implementation strategy:
- Requirement analysis of role-based workflows.
- API-first backend structuring for independent module development.
- Progressive frontend integration with protected routes and role guards.
- Incremental testing for each module and cross-module interaction.
- Deployment-ready architecture with environment-driven configuration.

## 8. Technology Stack
- Frontend: React, Vite, React Router, TailwindCSS, Axios, Socket.IO Client
- Backend: Node.js, Express.js, JWT, Mongoose, Socket.IO
- Database: MongoDB
- Cloud/External Services: Razorpay, Cloudinary, Web Push (VAPID), Email Service
- AI Processing: Local AI service for question generation from PDF content

## 9. Functional Modules Developed
- Authentication and Role Management
- User Profile and Teacher Verification
- Board/Class/Subject/Topic Management
- Question Bank and Multi-type Question Creation
- Exam Attempt and Results Flow
- Self Study (AI-generated questions from uploaded PDF)
- Study Material Upload, Purchase, and Secure Access
- Package and Subscription Management
- Wallet, Coins, Redemption, and Gift Card Inventory
- Notification and Push Notification System
- Real-time Chat Module
- CMS-style Content Management for public pages
- Promotion Scheduler and Manual Promotion Controls

## 10. Roles and Responsibilities
### 10.1 Student
- Access learning content, exams, self-study, results, materials, and packages.
- Purchase materials/packages and manage subscription benefits.
- Receive notifications and engage via dashboard tools.

### 10.2 Teacher
- Participate in academic workflows and content operations.
- Access student-related academic views as allowed.
- Use verified account flow for sensitive operations.

### 10.3 Admin
- Full operational control across users, academic masters, content, commerce, and system tools.
- Manages notifications, gift cards, packages, purchases, subscriptions, and promotions.

## 11. Key Outcomes
- Built a production-oriented full-stack educational platform architecture.
- Implemented secure and scalable role-based workflows.
- Delivered integrated commerce and engagement modules in an EdTech product.
- Applied AI-based automation for question generation and self-practice workflows.
- Established a maintainable modular codebase suitable for future expansion.

## 12. Learning and Internship Contribution
This internship strengthened practical understanding of:
- Real-world full stack architecture design
- API security and role-based authorization
- Payment and cloud integration patterns
- Data modeling for complex educational workflows
- Building scalable and maintainable module-driven systems
