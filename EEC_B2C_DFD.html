1. DFD Overview

This document defines the Data Flow Diagram for EEC B2C at Context (Level 0) and Level 1. It represents how external entities interact with the platform processes, data stores, and integrated third-party services.

2. External Entities

Entity IDExternal EntityDescription
E1Public UserUnauthenticated visitor browsing public pages and newsletter.
E2StudentAuthenticated learner using exams, materials, subscriptions, self-study.
E3TeacherAcademic user managing select content and student views.
E4AdminSystem operator managing full platform lifecycle.
E5RazorpayPayment gateway for orders, verification, and webhook events.
E6CloudinaryStorage service for uploaded files/media.
E7AI ServiceQuestion generation service from PDF text/content.
E8Email ServiceTransactional email delivery service.
E9Web Push ServicePush subscription and browser notification delivery.

6. Balancing Note

The Level 1 decomposition preserves the primary external flows from Level 0 by routing user/system interactions through specialized processes P1-P8 while maintaining integrated services (payment, storage, AI, email, push) and centralized persistence via MongoDB-backed stores.
